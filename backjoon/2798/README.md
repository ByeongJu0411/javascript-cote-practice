2798번 문제 풀이
이 문제는 n장의 카드 중에 3장을 골라 합이 M이 넘지 않으면서 M에 최대한 가까운 값을 출력합니다.

저는 3장의 카드를 고르는 모든 경우의 수를 완전 탐색으로 확인했습니다.
카드의 수는 최대 100이고 for문은 3중으로 구성했습니다.

제가 생각하는 핵심은 서로 다른 3장을 선택하는 것입니다.
이를 위해 다음과 같이 인덱스가 i < j < k를 만족하도록 반복문 범위를 설정했습니다.

for (let i = 0; i < n; i++) {
for (let j = 0; j < n; j++) {
for (let k = 0; k < n; k++) {
const total = cards[i] + cards[j] + cards[k];
if (total <= m) {
result = Math.max(result, total);
}
}
}
}

그래서 result를 합이 M 이하 경우에 최종적으로 가장 큰 값을 출력되게 했습니다.

하지만 1차 제출 때 에러가 생겼습니다.
코드를 분석해보니,

for (let i = 0; i < n; i++)
for (let j = 0; j < n; j++)
for (let k = 0; k < n; k++)

이 경우에 i, j, k가 같은 경우가 생긴다는 것을 발견했습니다.

예를 들어서 카드가 [1, 2, 3, 4, 5]이고 M이 9라고 가정하면
cards[4] + cards[4] + cards[4] = 15가 나와 에러가 생깁니다.

그래서 중복을 없애기 위해
j = i + 1로, k = j + 1로 수정하여 작성했습니다.

그래서 2차 제출에서 i < j < k를 적용하여 서로 다른 3장이 선택 되도록 했습니다.
