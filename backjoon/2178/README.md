이 문제는 N×M 미로에서 (1, 1)에서 출발해 (N, M)까지 도달하는 데 필요한 최소 칸 수를 구하는 문제입니다. 시작 칸과 도착 칸도 포함합니다.

1.  풀이
    - 최단 거리 문제이므로 BFS 로 풀었습니다.

          - 큐에 [x, y, dist] 형태로 좌표와 거리를 같이 저장했습니다.
          - 시작점 (0, 0) 부터 인접한 칸을 차례로 큐에 넣었습니다.
          - visited 배열로 이미 방문한 칸은 다시 안 넣었습니다.
          - 도착점 (N-1, M-1) 에 처음 도달한 순간의 dist 가 곧 최단 거리로 됩니다.

2.  오류 - 처음부터 BFS로 풀지 않았고, 단순한 발상에서 시작해서 여러 번 고쳐나갔습니다.

    2-1. 1차 풀이
    "재귀로 상하좌우를 다 호출하면 결국 도착점까지 가지 않을까?" 라는 발상으로 visited 없이 그냥 재귀를 돌렸습니다.
    → RangeError: Maximum call stack size exceeded. 옆 칸끼리 (0, 0) ↔ (0, 1) 무한 왕복 때문에 스택이 터졌습니다.

    2-2. 2차 풀이
    visited 배열을 추가해서 방문한 칸은 다시 안 가게 막았습니다.
    → 무한 루프는 풀렸는데 답이 정답보다 크게 나왔습니다. (예제 1: 17, 정답 15)
    한 번 방문 표시된 칸을 영영 못 들어가니까, 처음 도달한 한 경로만 보고 끝나서 더 짧은 경로를 놓치는 거였습니다.

    2-3. 3차 풀이
    재귀에서 빠져나올 때 visited[x][y] = false 로 풀어주는 백트래킹을 추가했습니다.
    → 예제 1~4 모두 정답을 통과했습니다. 그런데 20×20 전부 1인 미로를 직접 만들어 돌려보니 30초가 지나도 결과가 안 나왔습니다. 모든 경로를 다 나열하는 방식이라 칸 수에 대해 지수적으로 폭발합니다. 문제 조건은 N, M ≤ 100 이라서 이대로는 안 됩니다.

    2-4. 4차 풀이
    최단 거리니까 BFS 가 맞다고 판단하고 다시 짰습니다. 그런데 visited 처리를 큐에서 꺼낼 때 했습니다.
    → 답은 맞게 나옵니다. 그런데 코드를 다시 보니, (0, 1) 에서도 (1, 1) 을 큐에 넣고, (1, 0) 에서도 같은 (1, 1) 을 또 큐에 넣게 됩니다. 같은 칸이 큐에 여러 번 들어가서 큐 크기가 비효율적으로 부풀어 오릅니다. 또 Array.prototype.shift() 가 O(N) 이라 큐가 커질수록 매번 느려집니다.

    2-5. 5차 풀이 (최종)
    두 가지를 모두 수정했습니다.
