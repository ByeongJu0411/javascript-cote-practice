이 문제는 방향 없는 그래프가 주어졌을 때, 연결 요소(Connected Component)의 개수를 구하는 문제입니다.

## 1. 풀이

- DFS로 풀었습니다.
  - 1번부터 N번 정점을 순서대로 순회하면서, 아직 방문하지 않은 정점이 있으면 DFS를 호출합니다.
  - DFS가 한 번 호출될 때마다 해당 정점과 연결된 모든 정점을 visited 처리합니다.
  - DFS 호출이 끝날 때마다 answer를 1 증가시킵니다.
  - 결국 answer가 곧 연결 요소의 개수가 됩니다.

## 2. 오류

처음부터 바로 짜진 않았고, 몇 가지 실수를 겪었습니다.

### 2-1. 오타

코드를 작성하다가 `fs.readFileSync`를 `fs / readFileSync`로, `getInt`를 `getInut`으로 잘못 입력했습니다.
→ 런타임 에러가 발생했고, 오타를 수정한 뒤 정상 동작했습니다.

### 2-2. visited 초기화 누락

`visited`를 `[]`로만 선언했더니 `visited[i]`가 `undefined`를 반환해 논리적으로는 동작했지만, 의도가 불명확한 코드였습니다.
→ `Array(N+1).fill(false)`로 명시적으로 초기화했습니다.
