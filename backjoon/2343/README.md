이 문제는 블루레이 크기를 정해놓고 그 크기로 M개 이하의 블루레이에 다 담을 수 있는지 확인하면 되는 문제입니다.

1. 풀이

- left : 블루레이 크기의 최솟값
- right : 블루레이 크기의 최대값
- mid로 크기를 잡고 이 크기로 몇 개의 블루레이가 필요한지 세기
- M개 이하면 크기를 줄이고 초과하면 늘리기

countBluray 함수에 앞에서부터 순서대로 강의를 담다가 현재 블루레이가 못 넣게 되면 새 블루레이를 시작하는 식으로 풀이했습니다.

2. 오류
   1차 풀이에서 블루레이 크기를 최소로 하라 그래서 1로 잡았습니다. 하지만 mid가 5라고 가정하면, 강의 중에 9분짜리가 있을 때 그 강의를 블루레이 하나에 아예 못 담게 되는데, countBluray 함수에서 그냥 새 블루레이에 넣게 되버립니다. size보다 큰 강의가 들어가도 에러 처리를 안하게 되서 불가능한 크기를 가능한 크기로 판단해버리게 됩니다.

3. 해결
   left를 강의 중 최대값으로 바꿨습니다.

- let left = Math.max(...lectures);

4. 느낀점

- 범위 설정이 진짜 중요함을 다시 느꼈습니다. 로직은 맞는 듯 해도 값 하나를 잘못 잡으면 바로 틀린다는 것을 느꼈습니다.
